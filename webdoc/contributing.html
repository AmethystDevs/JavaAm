JavascriptAm Contribution Guide

Thanks for helping improve JavaAm! Follow these steps to contribute safely and effectively.

Reporting Bugs

  1. Search existing issues before reporting.
  2. Describe the problem clearly.
  3. Include exact reproduction steps.
  4. Share environment details: OS, Java version, Maven version.
  5. Attach error logs or screenshots if available.

Submitting Code

  1. Fork the repository and clone it locally.
  2. Create a focused branch: git checkout -b feature/my-feature
  3. Make your changes and keep commits small.
  4. Build and test the compiler locally.
  5. Push your branch and open a pull request.

Code Style

  • Use 4 spaces for indentation.
  • Follow camelCase for methods and variables.
  • Use PascalCase for class names.
  • Write clear comments and JavaDoc for public APIs.

Testing

cd compiler
mvn clean test
mvn test -Dtest=LexerTest

Test new language features and verify generated code compiles with javac.

Pull Request Checklist

  • Code follows project style guidelines
  • No new compiler warnings
  • Documentation updated
  • Tests pass locally
  • Commit messages are clear

Getting Help

If you need assistance, open an issue or discussion on GitHub.
